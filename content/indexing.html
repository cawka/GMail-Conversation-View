Indexing issue

Hi, this is Thunderbird Conversations!

We've been trying to setup Thunderbird Conversations to give you the best experience, but you have so many messages that reindexing them would crash your Thunderbird.

The tl;dr version

If you have an insanely high amount of non-bugmail email, delete the global-message-db.sqlite file in your profile folder (see link below).

With more explanations

You're in one of the two situations below.

Do you have a high amount of mail with attachments? Unless the first version of Thunderbird you ever tried is Thunderbird 3.3 alpha1 or later, these attachments won't show in the conversation view. We need to reindex all messages with attachments. The easiest way is for you to delete the global-messages-db.sqlite file in your profile folder.

Are you a Mozillian? We have a nice special-case for Bugzilla emails, where we replace the sender, i.e. bugzilla-daemon@mozilla.org, with the value of X-Bugzilla-Who header. You probably have way too much bugmail, and we need to reindex all of it to enable that feature. If you can live with the feature being enabled for newly arrived bugmail only, just forget about this message. Otherwise, you'll want to delete the global-messages-db.sqlite file in your profile folder.
